# Task: Audio Engine Core

## Description
Build the core audio engine using Web Audio API. Implement the AudioEngine class (master context + output chain with limiter), the Deck class (per-deck audio graph with source, 3-band EQ, volume), and the Crossfader. Support loading/decoding audio files (MP3, WAV, FLAC, OGG), play/pause/cue, pitch adjustment, and per-deck EQ control.

## Acceptance Criteria
- [ ] AudioEngine class creates and manages a single AudioContext
- [ ] Master output chain: merger → DynamicsCompressorNode (limiter) → destination
- [ ] Deck class with audio graph: AudioBufferSourceNode → BiquadFilterNode (lowshelf) → BiquadFilterNode (peaking/mid) → BiquadFilterNode (highshelf) → GainNode (volume) → output
- [ ] File loading via drag-and-drop and file picker, decoded with decodeAudioData()
- [ ] Supported formats: MP3, WAV, FLAC, OGG
- [ ] Play, pause, and cue (return to cue point) per deck
- [ ] Pitch/tempo adjustment via playbackRate (±8% range)
- [ ] 3-band EQ per deck: low (0-250Hz), mid (250Hz-4kHz), high (4kHz+)
- [ ] Volume fader per deck (0.0 to 1.0)
- [ ] Crossfader interpolation between deck outputs (linear, smooth, sharp curves)
- [ ] No audio clipping on master output

## Technical Details
- AudioEngine is a singleton managing the AudioContext lifecycle (resume on user gesture)
- Deck stores the decoded AudioBuffer and manages its own audio graph nodes
- EQ uses three BiquadFilterNodes: lowshelf (250Hz), peaking (1kHz, Q=1), highshelf (4kHz)
- Crossfader: two GainNodes, one per deck, with complementary gain curves feeding into merger
- Pitch adjustment via AudioBufferSourceNode.playbackRate (0.92 to 1.08)
- Cue point stored as a time offset; cue returns playback to that position

## Dependencies
None — uses browser-native Web Audio API only.
