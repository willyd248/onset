# Task: MIDI Controller Integration

## Description
Implement Web MIDI API integration for the Hercules DJControl Inpulse 200 MK2. Build the MIDIController class for device detection, the Hercules MIDI mapping config, the MIDIRouter for dispatching messages to app actions, and LED feedback output.

## Acceptance Criteria
- [ ] MIDIController class requests and manages Web MIDI API access
- [ ] Auto-detects Hercules DJControl Inpulse 200 MK2 by device name
- [ ] Connection status events (connected, disconnected, permission denied)
- [ ] hercules-mapping.json maps all relevant MIDI CCs and notes to named actions
- [ ] Mapped controls: jog wheels (touch + rotation), pitch faders, EQ knobs (low/mid/high × 2 channels), volume faders, crossfader, play/pause buttons, cue buttons, headphone cue buttons, pads (at least 4 per deck)
- [ ] MIDIRouter receives MIDI messages and emits named events (e.g., 'deck-a:eq-low', 'deck-b:pitch')
- [ ] LED feedback: play state LEDs, pad colors via MIDI output
- [ ] Graceful handling when controller disconnects mid-session (event + UI notification)
- [ ] Jog wheel rotation correctly interpreted as forward/backward nudge

## Technical Details
- Web MIDI API: navigator.requestMIDIAccess() with sysex: false
- Hercules Inpulse 200 MK2 uses USB MIDI class-compliant (no drivers needed)
- MIDI mapping JSON structure: { controls: [{ name, type: "cc"|"note", channel, number, action, deck }] }
- MIDIRouter uses an EventTarget or custom event emitter pattern
- LED output: send MIDI note-on messages to the output port with velocity for brightness/color
- Jog wheel: CC messages with relative encoding (64 = stopped, >64 = forward, <64 = backward)

## Dependencies
None — uses browser-native Web MIDI API.
