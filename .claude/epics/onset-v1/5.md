# Task: Waveform Data Processing

## Description
Build the WaveformData module that extracts peak amplitude data and frequency color information from decoded audio buffers. Generate multi-resolution peak data for overview and zoomed waveform views. Implement BPM detection using an onset/energy-based algorithm and beat grid generation.

## Acceptance Criteria
- [ ] WaveformData class takes an AudioBuffer and generates peak data
- [ ] Multi-resolution: overview (1 peak per ~100 samples) and zoomed (1 peak per ~10 samples)
- [ ] Peak data stored as Float32Arrays for memory efficiency
- [ ] Frequency coloring: FFT analysis on segments to classify as bass/mid/high dominant
- [ ] Color values stored alongside peaks (RGB tuples or hue values)
- [ ] BPMDetector analyzes AudioBuffer and returns estimated BPM (accurate ±1 BPM for 100-180 BPM range)
- [ ] Beat grid generated as array of time positions from BPM
- [ ] Processing completes in <3 seconds for a 5-minute stereo MP3 (44.1kHz)
- [ ] Results cached per loaded track to avoid recomputation

## Technical Details
- Peak extraction: iterate AudioBuffer channel data, find max absolute value per window
- Frequency analysis: run FFT (AnalyserNode or manual FFT) on segments, compute energy in bass (20-250Hz), mid (250-4kHz), high (4kHz-20kHz) bands
- BPM detection: energy-based onset detection → inter-onset intervals → histogram → peak = BPM
- Use OfflineAudioContext for non-real-time analysis if needed
- Store results in a Map keyed by a hash of the AudioBuffer (or file reference)
