# Task: Lesson Engine

## Description
Build the lesson engine — a state machine that loads lesson JSON files, renders step-by-step instructions in the lesson panel, validates user input from the MIDI controller and audio state, tracks progress, and persists completion state in localStorage.

## Acceptance Criteria
- [ ] LessonEngine class loads and parses lesson JSON files
- [ ] State machine: idle → active → step N → validating → (pass → next step | fail → retry) → complete
- [ ] LessonRenderer displays current step instruction text in the lesson panel
- [ ] Controller diagram SVG highlights the relevant physical control for each step
- [ ] InputValidator checks MIDI input and/or audio state against step validation criteria
- [ ] Validation types: pitch_match (BPM within tolerance), eq_value (knob in range), crossfader_position, beat_aligned (phase within tolerance)
- [ ] Progress bar showing step X of Y within current lesson
- [ ] Lesson completion stored in localStorage (lesson ID → completed timestamp)
- [ ] Lesson list shows completed/incomplete state per lesson
- [ ] "Next Step" and "Retry" controls available
- [ ] Transition quality score calculated on lesson 4 (beatmatching + EQ + crossfade combined)

## Technical Details
- Lesson JSON schema: { id, title, description, requiredTracks: [{deck, bpm, genre}], steps: [{id, instruction, highlight: "deck-a:pitch-fader", expected: {type, params}, validation: {type, tolerance}}] }
- State machine: simple switch-based or class with enter/exit per state
- InputValidator subscribes to MIDIRouter events and AudioEngine state
- Controller diagram: SVG with named groups per control, highlight = add CSS class to group
- localStorage key: "onset:lessons:{id}" → { completed: true, timestamp, score }
- Score calculation for transitions lesson: weighted average of beat alignment (40%), EQ smoothness (30%), crossfader curve (30%)

## Dependencies
- Task 002 (Audio Engine) — audio state for validation
- Task 003 (MIDI Controller) — MIDI input for validation
- Task 005 (Waveform Rendering) — beat grid for beat alignment validation
