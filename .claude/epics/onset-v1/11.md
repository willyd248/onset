# Task: Integration & Free Play Mode

## Description
Wire all modules together into the complete application. Implement the free play mode (full dual-deck interface without lesson overlays). Add performance tips, final polish, and end-to-end testing of the full user flow from controller connection to completing a transition.

## Acceptance Criteria
- [ ] App initializes correctly: creates AudioEngine, MIDIController, WaveformRenderers, LessonEngine
- [ ] Controller connection triggers UI updates across all components
- [ ] Loading a track on a deck: updates audio engine, generates waveform, displays in UI
- [ ] All MIDI controls route to correct audio parameters and update virtual mixer UI
- [ ] Free play mode: full interface, no lesson panel, optional performance tips as non-intrusive toasts
- [ ] Performance tips: 5-10 contextual hints (e.g., "Try adjusting the EQ before crossfading" when crossfader moves without EQ changes)
- [ ] Mode switching: Lessons ↔ Free Play preserves loaded tracks and playback state
- [ ] Error handling: graceful messages for unsupported browser, no MIDI, file decode failures
- [ ] Console is clean (no errors or warnings in normal operation)
- [ ] End-to-end flow works: plug in controller → app detects → load tracks → complete lesson → free play

## Technical Details
- main.js orchestrates initialization order: AudioEngine first (needs user gesture), then MIDIController, then UI components
- Event bus or direct wiring between modules (keep it simple — direct references, no framework)
- Performance tips: array of {condition, message} objects checked periodically
- Error overlay: full-screen message for critical failures (no Web MIDI support)
- Browser check: test for navigator.requestMIDIAccess before app init
