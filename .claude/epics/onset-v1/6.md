# Task: Waveform Canvas Rendering

## Description
Build the WaveformRenderer that draws real-time waveforms on HTML5 Canvas elements. Implement both the overview waveform (full track, static with moving playhead) and the zoomed waveform (~10 seconds around playhead, scrolling). Add beat grid overlay, frequency-based coloring, and phase alignment indicator between decks.

## Acceptance Criteria
- [ ] WaveformRenderer class manages two Canvas elements per deck (overview + zoomed)
- [ ] Overview waveform: full track rendered once, playhead drawn as a moving vertical line
- [ ] Zoomed waveform: ~10 seconds centered on current playback position, scrolls in real time
- [ ] Frequency-based coloring: waveform bars tinted by bass (warm/red), mid (green), high (cyan)
- [ ] Beat grid: vertical markers at beat positions overlaid on both waveform views
- [ ] Phase alignment indicator: visual cue showing beat alignment between Deck A and Deck B
- [ ] 60fps rendering using requestAnimationFrame
- [ ] Canvas resizes correctly with window
- [ ] Deck A styled with cyan (#00f0ff) accents, Deck B with magenta (#ff00e5)
- [ ] Glow effects on waveform (subtle outer glow matching deck color)

## Technical Details
- Overview: pre-render full waveform to an offscreen canvas, blit to visible canvas, draw playhead on top
- Zoomed: compute visible sample range from playback position, draw peaks from WaveformData
- Beat grid: draw vertical lines at beat positions (semi-transparent white or accent color)
- Phase indicator: compare beat positions of both decks, show offset as a bar or numeric display
- Use requestAnimationFrame loop shared between both decks for efficiency
- Canvas resolution: match devicePixelRatio for crisp rendering on HiDPI displays
