# Task: Lesson Content Creation

## Description
Author the 4 lesson JSON files that form the curriculum: (1) Setup & Basics, (2) Beatmatching, (3) EQ Mixing, (4) Transitions. Each lesson defines steps with instructions, expected controller inputs, and validation criteria. Also create the controller diagram SVG with named control groups for highlighting.

## Acceptance Criteria
- [ ] 01-setup.json: 6-8 steps covering MIDI connection, loading tracks, play/pause, cueing, basic navigation
- [ ] 02-beatmatching.json: 8-10 steps covering tempo display, pitch fader adjustment, jog wheel nudging, beat alignment verification
- [ ] 03-eq-mixing.json: 6-8 steps covering EQ band explanation, low/mid/high adjustment, bass swap technique
- [ ] 04-transitions.json: 8-12 steps combining all skills — cue next track, beatmatch, begin transition, EQ swap, crossfade, complete
- [ ] Each step has: clear instruction text (beginner-friendly language), highlight reference (which control), expected input, validation criteria
- [ ] Controller diagram SVG created with named groups: deck-a:play, deck-a:cue, deck-a:jog, deck-a:pitch, deck-a:eq-low, deck-a:eq-mid, deck-a:eq-high, deck-a:volume (same for deck-b), crossfader
- [ ] Lessons reference specific track requirements (BPM range, genre hints) for practice material
- [ ] Transitions lesson includes scoring criteria in the final step

## Technical Details
- JSON files placed in src/lessons/data/
- SVG: simplified top-down view of the Hercules Inpulse 200 MK2 layout with interactive groups
- Instruction text: short, action-oriented sentences ("Turn the pitch fader on Deck A slowly to the right")
- Validation tolerances: pitch match ±0.5 BPM, EQ values ±10%, beat alignment ±50ms
- Track requirements: setup lesson uses any tracks, beatmatching needs two tracks within ±10 BPM original, EQ needs tracks with distinct bass patterns, transitions uses tracks from prior lessons
