# Task: Virtual Mixer UI

## Description
Build the virtual mixer panel that mirrors the physical controller state. Render EQ knobs, volume faders, and crossfader as interactive, visually styled UI elements. Connect them bidirectionally to the audio engine and MIDI controller — physical changes update the UI, and UI changes update the audio.

## Acceptance Criteria
- [ ] Mixer panel rendered in the center section of the app
- [ ] Per-deck: 3 EQ knobs (low/mid/high), 1 volume fader
- [ ] Crossfader spanning both decks
- [ ] All controls styled as futuristic knobs/faders (dark background, glowing accent borders)
- [ ] Deck A controls use cyan glow, Deck B controls use magenta glow
- [ ] Controls respond to mouse drag interaction
- [ ] MIDI input updates control positions in real time
- [ ] UI control changes update the audio engine parameters
- [ ] Knob rotation shows value (0-100% or dB display)
- [ ] Crossfader position indicator with smooth animation
- [ ] Play/pause and cue buttons per deck with LED-style on/off states

## Technical Details
- Knobs: CSS + Canvas hybrid — circular knob graphic with rotation transform based on value
- Faders: styled range inputs or custom div-based sliders with drag handling
- Bidirectional binding: MIDI events update a shared state object, UI reads from and writes to the same state
- State management: simple observable store (Object with getters/setters that notify listeners)
- Animations: CSS transitions for smooth visual updates, no JS animation needed for controls
- Glowing borders: box-shadow with accent color, increased intensity on interaction

## Dependencies
- Task 002 (Audio Engine) — knobs/faders control audio parameters
- Task 003 (MIDI Controller) — physical controls update virtual controls
