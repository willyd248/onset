# Onset — Handoff Document

## What is Onset?
A DJ learning app that teaches mixing fundamentals through a Duolingo-inspired learning path. Built for the Hercules DJControl Inpulse 200 MK2 MIDI controller. Vanilla JS, no framework, Vite bundler.

## Architecture

### Core Modules (do not modify without understanding the full chain)
- **AudioEngine** (`src/audio/AudioEngine.js`) — Web Audio API singleton, master output chain
- **Deck** (`src/audio/Deck.js`) — Per-deck audio graph: source → gain → EQ → volume → output
- **Crossfader** (`src/audio/Crossfader.js`) — A/B blending with curve modes
- **MIDIController** (`src/midi/MIDIController.js`) — Web MIDI API, auto-detects Hercules controller
- **MIDIRouter** (`src/midi/MIDIRouter.js`) — Parses MIDI → named events
- **hercules-mapping.js** (`src/midi/hercules-mapping.js`) — Hardware-verified CC/note mappings

### State Management (the critical routing layer)
- **MixerState** (`src/ui/MixerState.js`) — Observable store, dispatches change events with source tracking
- **MixerBridge** (`src/ui/MixerBridge.js`) — Routes: State → Audio, MIDI → State, Deck events → State
- **MixerUI** (`src/ui/MixerUI.js`) — Bidirectional DOM ↔ State binding

### Data Flow
```
UI input → MixerState.set('ui') → MixerBridge → Deck methods
MIDI input → MIDIRouter events → MixerBridge → MixerState.set('midi') → MixerUI updates DOM
Deck events → MixerBridge → MixerState.set('audio') → MixerUI updates DOM
```

### Lesson Engine (`src/lessons/`)
- **LessonEngine** — State machine: idle → session → watch → imagine → do → scored → complete
- **LessonRenderer** — Renders phases, Guitar Hero feedback overlays on mixer controls
- **InputValidator** — Validates MIDI input against lesson targets
- **ScoringEngine** — Calculates accuracy/timing/phrase scores
- **SpacedRepetition** — Cross-session technique scheduling
- **SessionManager** — Creates lesson playlists

### UI Layer
- **App.js** — Main controller, initialization order, wiring
- **AppShell.js** — Mode switching (lessons/freeplay), drag-drop, MIDI status
- **ViewManager.js** — Multi-view routing (Learn, Practice, Stats, Profile, Settings)
- **WaveformRenderer** (`src/visuals/`) — Canvas-based real-time waveform rendering

## Key Element IDs
All JS modules query DOM elements by ID. These IDs are the contract between HTML and JS:
- Range inputs: `gain-a/b`, `eq-high/mid/low-a/b`, `volume-a/b`, `pitch-a/b`, `filter-a/b`, `crossfader`
- Buttons: `play-a/b`, `cue-a/b`, `sync-a/b`, `load-a/b`, `pad-1/2/3/4-a/b`
- Canvases: `waveform-a/b`
- File inputs: `file-a/b` (hidden)
- Status: `midi-status`, `streak-badge`, `streak-count`
- Lesson: `lesson-content`, `lesson-progress`
- Views: `view-practice`, `view-learn`, `view-stats`, `view-profile`, `view-settings`, `view-session-complete`
